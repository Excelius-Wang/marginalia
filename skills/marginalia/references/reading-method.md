# 阅读与分析流程（Reading Method）

这是精读的**内部流程**，不是输出结构。流程的产物按 `note-schema.md` 组织成笔记。目标是重建作者的思考路径、讲清方法、评估证据、找出脆弱假设，而不是复述 abstract。

## 1. 解析来源并通读

先按 `source-resolution.md` 拿到全文。拿到后，先完整读一遍 abstract、introduction、related work、method、experiments、conclusion，必要时读 appendix。

开始写之前，先确定三件事：

1. 论文的核心技术 claim 是什么。
2. 支撑这个 claim 的关键实验或关键论证是什么。
3. 最重要的 baseline 或 prior work 对比是什么。

只能拿到 abstract 时，必须说明分析是临时的，并指出哪些部分无法验证。

## 2. 三遍阅读

- **第一遍（问题与主张）：** abstract、introduction、conclusion。识别问题、本质动机、basic idea、声称的贡献、主结果。不要停在表面动机——区分真正的痛点、缺失的能力、被打破的假设，或让这篇论文值得做的研究 gap。
- **第二遍（方法与机制）：** method、threat model、假设、算法、系统设计、定义。提取论文真正的机制，而不是只在高层 paraphrase。重建 basic idea 如何从动机推出，再如何发展成具体方法。
- **第三遍（证据）：** evaluation、表、图、ablation、case study、limitations、related work。核对实验是否真的支撑 claim。

## 3. 校准 novelty

找 2–3 篇最接近的 prior work，定位这篇论文真正的 delta。区分方法新意、设定新意、数据新意、发现新意。不要凭记忆或感觉宣布它 novel，也不要为了凑 follow-up idea 而编造相关工作。如果 delta 依赖近期工作或发表后的影响，在当前这一轮里 web 搜索并引用来源，不要凭记忆引用影响力。

## 4. 重建作者思路（招牌）

在正式讲方法之前，逆向重建作者可能的思考路径。

**关键约束：** 这一步**不能**拿论文自己的贡献当前提。只能用这篇论文之前已经存在的背景、失败模式、经验观察、理论线索和相关工作。目标是模拟一个认真研究这个问题的人，如何从已有知识一步步走到这篇论文的 idea。

这一步逼你理解"为什么是这个方法"，而不是"这个方法是什么"。

## 5. 批判性综合（内部分析步骤）

把论文当成需要被质疑的对象，问这些核心问题：

- 在解决什么问题？之前的实践为什么没解决？什么是新的？谁受益？
- 会出什么错？复现或部署的成本是多少？实验是否真的测了这个 claim？
- 方法是否真的从动机推出，还是动机更多是事后包装（post-hoc framing）？

把判断落到具体处：不成立的威胁模型、不现实的假设、弱 baseline、窄数据集、缺失的 ablation、脆弱的标签、数据泄漏、事后规则、复现 gap、部署成本，或不清晰的真实影响。

这是**内部分析步骤**，不要在笔记里单开一节"批判性评审"。把判断转化成笔记里的 `毒舌评论`、`最脆弱的假设`、`最强反例设计`、`Strengths`、`Limitations`、`My Takeaways`。

## 6. 读数复核（写完后必做）

笔记最值钱的部分（`毒舌评论`、`最脆弱的假设`、`最强反例`）往往挂在几个具体数字上——一旦把表读串行/读错列，整段批判就立不住。写完后回到原文，逐一核对笔记里引用的每个关键数字（表格分数、提升幅度、token 数、超参）确实对得上来源的那一格，尤其是支撑批判结论的对照数（如"长度受控后打平"的两个 LC WR）。核对方式要落到具体：指明它来自哪张表的哪一行哪一列。拿不准的数字就标不确定或删掉，绝不让一个读错的数字撑起一句狠话。

## 7. 全程信息来源纪律

整个流程中，严格区分四类信息（细则见 `style-guide.md`）：论文原文明确声称 / 相关文献中的已有结论 / 基于证据的合理推断 / 仍然不确定的猜测。不要把推断写成事实，不要把猜测写成推断。
